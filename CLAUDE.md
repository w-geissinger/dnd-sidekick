# CLAUDE.md — DnD Sidekick

Browser-based D&D 5e character sheet manager. No backend — all data lives in `localStorage`. Built with React 19, MobX 6, TanStack Router (file-based), Tailwind CSS 4 + daisyUI 5, and Vite 6.

---

## Commands

```bash
pnpm install       # Install dependencies
pnpm dev           # Dev server on http://localhost:3001
pnpm build         # Production build
pnpm typecheck     # tsc --noEmit (no test runner configured)
pnpm serve         # Preview production build
```

Package manager is **pnpm** (lockfile: `pnpm-lock.yaml`).

## State Management (MobX)

All stores use `makeAutoObservable()`. Components must be wrapped with `observer()` from `mobx-react-lite`.

### Store Access

```ts
import { useCharacterStore, useReferenceDataStore, useUIStore, useRootStore } from '../stores/RootStore';
```

### CharacterStore

- `characters: Character[]` — full list
- `activeCharacterId: string | null`
- `activeCharacter` (computed) — the current character
- Key actions: `createCharacter`, `updateCharacter`, `updateActiveCharacter`, `deleteCharacter`
- Spell actions: `addSpellToCharacter`, `toggleSpellPrepared`, `removeSpell`, `addAttackFromSpell`
- Persistence: auto-saves to `localStorage` key `dnd-sidekick-characters`
- Export: `exportCharacter(id)`, `exportAllCharacters()`, `importCharacters(json)`

### ReferenceDataStore

- Wraps the `dnd-data` npm package (5,000+ spells, 15,000+ items)
- Actions: `searchSpells(query, level?, className?)`, `searchItems(query, type?)`, `searchBackgrounds(query)`
- Returns top 100 results, sorted: exact match → starts with → contains
- `skillsForNewCharacter()` — builds default 18-skill array

### UIStore

- `activeModal: 'newCharacter' | 'spellSearch' | 'itemSearch' | null`
- `activeCharacterSheetTab: 'combat' | 'spells' | 'equipment' | 'features' | 'lore'`
- `theme: string` — one of 32 daisyUI themes, persisted to localStorage
- `spellColumns` / `itemColumns` — visible column config, persisted

## Routing (TanStack Router)

File-based routing — **`routeTree.gen.ts` is auto-generated by the Vite plugin** on dev/build. Never edit it manually.

| Route | File | Notes |
|---|---|---|
| `/` | `routes/index.tsx` | Landing page |
| `/character/$characterId` | `routes/character.$characterId.tsx` | Dynamic route param |
| `/about` | `routes/about.tsx` | Static page |

To navigate programmatically use `useNavigate` / `Link` from `@tanstack/react-router`.

## Key Utility Functions (`utils/characterUtils.ts`)

```ts
getAbilityModifier(score: number): number   // Math.floor((score - 10) / 2)
getProficiencyBonus(level: number): number  // Math.ceil(level / 4) + 1
formatModifier(mod: number): string         // "+3" or "-1"
buildDefaultCharacter(draft, skills)        // Full Character with sensible D&D defaults
generateId(): string                        // crypto.randomUUID()
```

Hit die defaults per class: Barbarian d12 | Fighter/Paladin/Ranger d10 | Sorcerer/Wizard d6 | others d8.

## Character Data Shape (abbreviated)

```ts
interface Character {
  id: string;
  name: string; class: string; race: string; background: string;
  level: number; exp: number; alignment: string;
  abilityScores: AbilityScores;          // { str, dex, con, int, wis, cha }
  savingThrows: SavingThrow[];           // proficiency toggles
  skills: CharacterSkill[];              // 18 skills, proficiency + expertise
  armorClass: number; initiative: number | null; speed: number;
  hp: { max: number; current: number; temp: number };
  hitDice: string; deathSaves: DeathSaves;
  attacks: Attack[];
  equipment: EquipmentItem[]; currency: Currency;
  spells: CharacterSpell[]; spellSlots: Record<string, SpellSlot>;
  spellcastingAbility: AbilityScoreKey | '';
  spellSaveDC: number; spellAttackBonus: number; maxPreparedSpells: number;
  // ...personality, backstory, proficiencies, languages, features, etc.
  createdAt: string; updatedAt: string;
}
```

## Styling

- **Tailwind CSS v4** — utility classes, no config file needed (v4 uses CSS-native config via `@import`)
- **daisyUI v5** — component classes (`btn`, `modal`, `card`, `tabs`, etc.) + 32 themes
- Theme is set via `data-theme` attribute on `<html>` and persisted by UIStore
- Icons: `react-bootstrap-icons` — import named icons like `import { PlusCircle } from 'react-bootstrap-icons'`

## Gotchas & Patterns

- **`routeTree.gen.ts`** is regenerated automatically — any manual edits will be overwritten.
- **`initiative`** defaults to `null` and auto-calculates from DEX modifier in the UI — don't store a computed value as source of truth.
- **`updateActiveCharacter(patch)`** is a shallow merge — only pass changed fields.
- **`observer()`** wrapper is required on any component that reads MobX observables, or it won't react to changes.
- **`SectionCard`** is the standard wrapper for all character sheet sections — use it for consistency.
- **localStorage** is the only persistence layer. No network calls. Export to JSON for backup.
- Adding a new route: create a file in `src/routes/` following TanStack Router naming conventions; the route tree updates on next dev server start.
